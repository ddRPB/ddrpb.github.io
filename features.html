Features

RPB portal is the principal component of the platform that provides fundamental data management features for efficient conduction of multi-centre clinical studies. RPB portal alone serves as an integration engine that handles communication between the core components but also extends the platform about new dedicated features.

Key features of RPB portal

  • Manage trials with basic Clinical Trial Management System (CTMS)
  • Pseudonymise patients participating in multi-centre clinical trial
  • Randomise patients in randomised trials
  • Track patient participation across multiple studies
  • Manage collection of medical imaging and treatment plans in DICOM format
  • Handles DICOM data de-identification and RTSTRUCT naming harmonisation
  • Enables biospecimen data collection in clinical trials (*link to BIO-banking system planned)
  • Allows semantic annotation of CRF forms (*link to metadata repository planned)
  • Provides automatic patient registration for patient registries based on HL7 stream
  • Enables patient reported outcomes (ePRO) and clinician outcome assessment (eCOA) data collection
  • Permission based security for user authorisation
  • Provides REST based web services for software system integration

One of the core design principle of RPB platform design was data interoperability. The platform in general tries to reuse well accepted vendor neutral data formats and standards.

Standards

  • XLS templates for CRF definitions
  • XForms CRF representations
  • OpenRosa API for XForms
  • CDISC ODM, SPSS, CSV study data exports
  • DICOM and DICOM-RT data formats
  • DICOM WADO access to DICOM data
  • DICOMweb support (*planned)
  • DICOM Supplement 142 for Clinical Trial De-identification
  • HL7 communication bridge
  • LDAP for user authentication (*planned)
  • Internally using SOAP and REST based web services
