Components

The complex nature of radiotherapy research data requires from RPB to deal with data management within different scopes/sub-domains and integrate with several clinical or research IT systems.

Work Packages Diagram

WP0 - link to clinical systems

  • HL7 - NextGen Connect healthcare communication and monitoring engine

WP1 - patient privacy

  • PID - Mainzelliste patient identity management and pseudonym generation

WP2 - clinical research

  • CTMS - RPB portal providing integrated access to the platform
  • ePRO - RPB mobile with Enketo tablet friendly XForms
  • EDC - LibreClinica electronic data capture system
  • PROMAN - Redmine issue tracking system

WP3 - medical imaging

  • PACS - Conquest research DICOM data store
  • Uploader - RPB uploader for web based de-identification and upload of DICOM data
  • Viewer - Weasis standalone DICOM-RT viewer
  • Processor - Clinical Trials Processor for DICOM pipelines orchestration

WP4 - microscopy imaging

  • WSI - DICOM whole slide imaging planned only as third party product integration

WP5 - biospecimen data

  • BIO - third party commercial product integration

WP6 - laboratory assays data

  • LAB - LabKey file repository and data tabulation

WP7 - exploration & reporting

  • LAB - study data reload and cross component data aggregation, search and reporting

WP8 - platform deployment

  • Depending on trials unit available infrastructure on a site. Recommended is normally virtual machine and container based deployment with dedicated reverse web proxy in demilitarised zone (DMZ).

WP9 - link to national and international research platforms

  • DKTK CCP-IT - project specific pseudonymisation
  • DKTK JIP - data loading for analytical processing

WP10 - learning database

  • LAB - scripting and API for machine learning purposes
