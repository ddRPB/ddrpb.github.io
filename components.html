Components

RPB tries to tackle with very complex data management tasks that need to be mastered to successfully support translation research projects in radiotherapy. It would not be feasible to have a super complex system that covers requirements from all types of potential end users in translational research environment. That is why RPB platform is using a carefully chosen set of components that are meant to solve certain data management tasks. RPB portal than integrate all these together to provide a single staring dashboard for all types of end-users. To provide a good overview the RPB components are sorted into work packages.

WP0 - link to clinical systems

  • HL7 listener - Mirth Connect

WP1 - patient privacy

  • Patient identity database and pseudonym generation Mainzelliste

WP2 - clinical research

  • EDC - OpenClinica
  • EDC - web services
  • EDC - rule designer
  • XForms render engine - ENKETO
  • RPB portal - integration engine
  • RPB participate - mobile data entry portal

WP3 - medical imaging

  • Research PACS - Conquest
  • RPB client - de-identification and upload of DICOM data
  • DICOM WADO web viewers - Java webstart Weasis, HTML5 DWV

WP4 - microscopy imaging (*planned)

WP5 - bio specimen data (commercial product Centraxx)

WP6 - molecular profiling data (*planned)

WP7 - clinical research data-warehouse (*planned)

  • tranSMART

WP8 - platform deployment

  • supporting systems for RPB development and deployment needs (github, continuous integration server, web presence, para virtualisation with xen, backup strategy, reverse proxy nginx in DMZ for external access, etc.)

WP9 - link to national and international research platforms

  • DKTK CCP-IT link to local bridgehead and others...
